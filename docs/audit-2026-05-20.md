## P0 — Critical (Block ALL Clinical Use)

### P0-A — Arabic user in crisis receives English-only response

**Audits:** 12.1b, 13.1  
**Evidence:** Crisis path is `safety_check → crisis_response → END` (confirmed from Mermaid graph). `output_gate` (the only node that calls `translate_to_arabic`) is bypassed. `CRISIS_RESPONSE` is an English-only hardcoded string. An Arabic speaker who writes `أريد الموت` gets a response they cannot read.

**Recommended fix:** Add a bilingual fallback to `_crisis_response_node` in `graph.py`:
```python
CRISIS_RESPONSE_AR = (
    "أنا قلق جداً على ما شاركته. "
    "يُرجى التواصل مع خط الأزمات فوراً — "
    "في الإمارات: توازن 800-4673، أو الرقم الدولي: 988 (الولايات المتحدة). "
    "أنت لست وحدك."
)

def _crisis_response_node(state: SageState) -> dict:
    lang = state.get("detected_language", "en")
    response = CRISIS_RESPONSE_AR if lang == "ar" else CRISIS_RESPONSE
    return {
        "is_safe": False,
        "active_skill_id": None,   # see P0-B
        "response": response,
        "response_en": CRISIS_RESPONSE,
        "path": state["path"] + ["crisis_response"],
    }
```

### P0-B — Active skill persists through crisis turn; CBT continues after suicidal disclosure

**Audits:** 13.4, 22.2  
**Evidence:** Live test confirmed: graph invoked with `active_skill_id="cbt_thought_record"` + crisis message `"I want to kill myself right now"` returns `active_skill_id="cbt_thought_record"` (unchanged). `_crisis_response_node` returns only `{is_safe, response, response_en, path}` — LangGraph merges the partial dict preserving `active_skill_id` from incoming state. `run.py` then carries `result.get("active_skill_id")` into the next turn. The next message from this user re-enters `skill_executor` at `explore_distortion`.

**Recommended fix:** `_crisis_response_node` must explicitly return `active_skill_id: None` and `active_step_id: None` (shown in fix above).

## P1 — High (Fix Before Any User-Facing Test)

### P1-1 — Missing critical suicidal ideation phrases in CRISIS_KEYWORDS

**Audit:** 3.1  
**Evidence:** Live test: `"I don't want to be alive"` → `is_safe=True`. `"end it all"` → `is_safe=True`. These common suicidal phrases are absent from `CRISIS_KEYWORDS`. The system marks them safe and routes to `intent_route`, which may or may not classify them as crisis. The hard safety net fails.  
**Also:** `"I had a glass of wine with dinner"` → `substance_use=True` (false positive — `"wine"` bare keyword matches social drinking).

**Recommended additions to `CRISIS_KEYWORDS`:**
```python
# English additions
"don't want to be alive", "don't want to live", "not worth living",
"end it all", "end my life", "take my own life", "take my life",
"jump off", "jump from", "hang myself", "overdose on",
"no reason to live", "can't go on", "want to disappear forever",
# Arabic additions (verify with native speaker)
"لا أريد العيش", "لا أستطيع الاستمرار", "أريد الاختفاء",
```
**Refine `substance_use`:** Replace bare `"wine"` and `"beer"` keywords with phrase-context patterns or require co-occurrence with distress markers.

### P1-2 — L1 exit phrases match ordinary sentences (false skill exits)

**Audit:** 3.2  
**Evidence:** Live test: `"I can't stop thinking about this"` → L1 exit fired. `"I've had enough sleep lately"` → L1 exit fired. `"I don't want to quit smoking"` → L1 exit fired (matched both "quit" and "don't want to"). `check_escalation` does bare substring matching: `any(phrase in msg for phrase in L1_EXIT_PHRASES)` where phrases include single words like `"stop"`, `"quit"`, `"enough"`, `"leave"`, `"exit"`.

**Recommended fix:** Replace single-word L1 phrases with whole-word or boundary-anchored patterns:
```python
import re
_L1_PATTERNS = [re.compile(r'\b' + re.escape(p) + r'\b', re.IGNORECASE) for p in [
    "stop this", "want to stop", "let's stop", "can we stop", "want to quit",
    "don't want to do this", "end this session", "leave now", "exit"
]]
def _is_l1_exit(msg: str) -> bool:
    return any(p.search(msg) for p in _L1_PATTERNS)
```

### P1-5 — Crisis exchange not added to conversation_history

**Audit:** 13.3  
**Evidence:** `_crisis_response_node` does not return `conversation_history`. LangGraph merge preserves the incoming history unchanged. The crisis message and response are invisible to subsequent LLM calls. On the next turn, the LLM has no context that a crisis just occurred — a user's follow-up "I still feel that way" arrives with no prior context.

**Recommended fix:** Add history append to `_crisis_response_node`:
```python
"conversation_history": state.get("conversation_history", []) + [
    {"role": "user", "content": state.get("message_en", "")},
    {"role": "assistant", "content": CRISIS_RESPONSE},
],
"turn_count": state.get("turn_count", 0) + 1,
```

### P1-6 — Ollama failure crashes entire session (language.py, output_gate.py)

**Audit:** 16.1a, 16.1c  
**Evidence:** `language.py` has zero try/except around `ollama.Client.chat()`. `output_gate.py` has none around `translate_to_arabic()`. Any Ollama network failure, model unload, or timeout propagates unhandled through `safety_check_node` or `output_gate_node`, crashing `graph.invoke()` with a raw traceback.

**Recommended fix (language.py):**
```python
def translate_to_english(text: str) -> str:
    try:
        response = client.chat(model=TRANSLATION_MODEL, messages=[...])
        return response["message"]["content"].strip()
    except Exception:
        return text  # fallback: return original on translation failure

def translate_to_arabic(text: str) -> str:
    try:
        response = client.chat(model=TRANSLATION_MODEL, messages=[...])
        return response["message"]["content"].strip()
    except Exception:
        return text  # fallback: return English response
```

### P1-7 — JSONDecodeError crash in intent_route when LLM returns malformed JSON in braces

**Audit:** 16.2  
**Evidence:** Live test: LLM response containing `{invalid: json}` (unquoted keys, common from some models) → `json.loads` raises `JSONDecodeError` with no surrounding try/except. Response of plain text (no braces at all) → safe fallback to defaults. The critical path: regex finds `{...}`, `json.loads` is called, JSON is invalid → crash.

**Recommended fix:**
```python
match = re.search(r'\{.*\}', raw, re.DOTALL)
data = {}
if match:
    try:
        data = json.loads(match.group(0))
    except json.JSONDecodeError:
        pass  # fall through to defaults below
```

### P1-8 — freeflow_respond sends system prompt as HumanMessage (wrong API role)

**Audit:** 15.1  
**Evidence:** `compose_prompt()` returns a single string starting with `"SYSTEM: You are Sage..."`. `freeflow_respond_node` calls `llm.invoke(prompt_string)`. LangChain's `BaseChatModel` wraps a bare string as a `HumanMessage` — the entire prompt including persona, clinical adaptations, and skill instructions arrives at the API in the user role. `intent_route_node` and `low_confidence_respond_node` both correctly use `[{"role": "system", ...}, {"role": "user", ...}]`.

**Impact:** Clinical adaptations and the no-diagnose persona are treated as user suggestions, not system constraints. They are easier to override via prompt injection. Instruction following is less reliable.

**Recommended fix:**
```python
def freeflow_respond_node(state: SageState, llm=None) -> dict:
    if llm is None:
        llm = get_responder()
    system_content, user_content = compose_prompt_parts(state)
    messages = [
        {"role": "system", "content": system_content},
        {"role": "user", "content": user_content},
    ]
    response = llm.invoke(messages).content.strip()
    return {"response_en": response, "path": state["path"] + ["freeflow_respond"]}
```
Refactor `compose_prompt` to return `(system_str, user_str)` instead of a single concatenated string.

### P1-9 — skill_select re-runs on new_skill intent mid-skill, silently discards progress

**Audit:** 2.2 (Row 7)  
**Evidence:** `_route_after_intent` routes `"new_skill"` → `"skill_select"` regardless of whether `active_skill_id` is set. `skill_select_node` may return `active_skill_id=None` (no keyword match) or a different skill. User's mid-session CBT progress is silently lost.

**Recommended fix:** When `active_skill_id` is set and intent is `"new_skill"`, route to `skill_executor` with a special "re-anchor" step, or surface a transition prompt:
```python
if intent == "new_skill":
    if state.get("active_skill_id"):
        return "skill_executor"  # stay in current skill, re-anchor
    return "skill_select"
```

### P2-2 — StopIteration crash if active_step_id doesn't match skill steps

**Audit:** 3.4, 16.1b  
**Evidence:** `skill_executor.py` line ~80: `step = next(s for s in skill.steps if s.step_id == current_step_id)` — bare `next()` with no default raises `StopIteration` on mismatch. Confirmed by live test with `"nonexistent_step_id"`.

**Fix:**
```python
step = next((s for s in skill.steps if s.step_id == current_step_id), None)
if step is None:
    # Log and fallback to first step
    return {"action": "advance", "next_step_id": skill.steps[0].step_id, "instruction": skill.steps[0].examples[0]}
```

### P2-5 — RTL Unicode control characters cause wrong language detection

**Audit:** 6.1c  
**Evidence:** Text with U+202B/U+202C RTL embedding markers → langdetect returns `"so"` (Somali) instead of `"en"`. The Arabic Unicode check passes correctly, but Latin-script normalization only catches pure ASCII.

**Fix:** Strip Unicode format characters (category `Cf`) before detection:
```python
import unicodedata
text_clean = ''.join(c for c in text if unicodedata.category(c) != 'Cf')
```

### P2-6 — Zero-width space (U+200B) bypasses crisis keyword detection

**Audit:** 17.2  
**Evidence:** `"kill​myself"` (with ZWSP) → `crisis=False`. `"kill myself"` → `crisis=True`. All 13 English crisis keywords bypassed by inserting U+200B between words.

**Fix:** Normalize before keyword matching:
```python
import unicodedata
msg_normalized = unicodedata.normalize('NFKC', msg).replace('​', '').replace('‌', '').replace('‍', '')
```

### P2-7 — type coercion failure if LLM returns string for emotional_intensity

**Audit:** 1.3  
**Evidence:** `intent_route_node` calls `int(data.get("emotional_intensity", 5))`. If the LLM returns `"moderate"` instead of `7`, `int("moderate")` raises `ValueError` with no try/except.

**Fix:**
```python
def _safe_int(val, default):
    try:
        return int(val)
    except (TypeError, ValueError):
        return default
emotional_intensity = _safe_int(data.get("emotional_intensity"), 5)
```

### P2-10 — CLI has no error handling (crash on any node failure)

**Audit:** 22.1  
**Evidence:** `run.py` has zero try/except. `Ctrl-C` crashes with traceback. `Ctrl-D` raises `EOFError`. Any Ollama or OpenRouter failure propagates raw.

**Fix:**
```python
def main():
    ...
    while True:
        try:
            user_input = input("You: ").strip()
        except (KeyboardInterrupt, EOFError):
            print("\nGoodbye.")
            break
        ...
        try:
            result = graph.invoke(state)
        except Exception as e:
            print(f"\n[Error] Something went wrong: {e}")
            print("[Sage] I'm having trouble right now. Please try again.")
            continue
```

### P2-11 — Loose dependency version bounds; no lock file

**Audit:** 11.1  
**Evidence:** `langgraph>=0.2.0` allows v0.2.x which has a fundamentally different API from v1.2.0 (installed). No `requirements.lock` or `uv.lock`. Fresh install on another machine may resolve to incompatible versions.

**Fix:** Tighten bounds: `langgraph>=1.0.0,<2.0.0`. Generate lock: `uv pip compile pyproject.toml -o requirements.lock`.

### P2-12 — Dual loading inconsistency: skill_select pre-loads, skill_executor re-reads disk per turn

**Audit:** 18.1  
**Evidence:** `skill_select._SKILLS` loads at module init. `skill_executor.load_skill(skill_id)` is called on every invocation (disk I/O + Pydantic parse per turn). Inconsistent; O(turns) disk reads.

### P2-14 — Terminal encoding — no UTF-8 guarantee for Arabic output

### P3-8 — Unbounded conversation_history growth in state

**Audit:** 22.3  
`output_gate` appends 2 entries per turn with no size cap. After 100 turns: 200 entries in state, all serialized per invocation. `freeflow_respond` uses only `[-4:]`. Add a cap:

history = (state["conversation_history"] + [...])[-40:]  # keep last 20 exchanges

## Appendix: Questions for Domain Expert / Clinical Advisor

1. **CRISIS_KEYWORDS completeness:** Are `"don't want to be alive"`, `"end it all"`, `"not worth living"` the right additions? What other idioms should be included for the UAE/Gulf population specifically?
2. **Arabic crisis keywords:** Are `أريد الموت`, `انتحار`, `أنهي حياتي`, `أؤذي نفسي` appropriate for Gulf-dialect Arabic? Are there dialectal variants that should be added?
3. **`emotional_intensity > 7` threshold:** Is this the right threshold for pausing CBT progression and switching to validation-only mode? Should it be lower (6) for a more cautious system?
4. **`engagement < 3` threshold:** Is asking a check-in question the right response to very low engagement, or should the system consider ending the session gracefully?
5. **L2 clinical flag persistence:** Should `clinical_flags` accumulate across turns (proposed) or reset per turn? What is the clinical implication of a "lingering" substance_use flag in subsequent turns that don't mention substance use?
6. **Araglish crisis phrases:** Should the system treat a mixed-script message like `"I can't take it anymore والله"` as Arabic (Arabic Unicode present → detected as `ar`) for crisis response language purposes?
7. **UAE crisis lines:** Confirm correct numbers: Tawazun 800-HOPE (4673) — is this currently active? Is there a better 24/7 line for mental health specifically?
