## Task 2 — STATE-5: Fix sign-out PII leak
**Stream 1 only.**

**Context:** `signOutUser` in `lib/auth-actions.ts` calls `supabase.auth.signOut()` and redirects. It never clears the Zustand `useOnboardingStore`, which is persisted to `localStorage` under the key `cdai-onboarding`. On a shared device the next user's sign-up flow is pre-filled with the previous user's `name`, `ageRange`, `role`, and `wellnessQ1`/`wellnessQ2` answers. The store has a `reset()` action — we call it before signOut.

## Task 3 — STATE-2 + STATE-3: Add Zod validation to API routes
**Stream 1 only.**

**Context:** `/api/chat` accepts unvalidated JSON. `messages.content` is interpolated directly into the LLM classifier prompt — a prompt injection vector. `sessionId` is passed to the ownership DB query without UUID validation. `/api/feedback` similarly accepts `messageId` without UUID validation. Zod v4 is already installed.

**Architecture note (do not change, just understand):** `crisisState`, `clinicalFlags`, and `distressTrajectory` are ferry fields — the sage-poc backend emits them as `X-Sage-*` response headers, and the client forwards them back on the next request to preserve conversational continuity across turns. The sage-poc backend (Node 1, `safety_check`) is the authoritative source for crisis detection; these client-supplied values are hints for state continuity, not inputs to safety logic. Zod validation here prevents malformed data, but the backend must never trust these fields for safety decisions. A code comment is added to the route to make this explicit.

## Task 5 — PWA-1 + PWA-2: Create and register the service worker; fix update race condition
**Stream 2 only.**

- [ ] **Step 5.1: Create `apps/web/public/sw.js`**

```js
// Service worker for Sage — minimal offline fallback.
// POST-PILOT: Replace with Serwist (@serwist/next) for full precaching and
// background sync. The source for this file is sw.ts in the repo root.
const CACHE_NAME = 'cdai-v1'
const OFFLINE_URL = '/offline.html'

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => cache.add(OFFLINE_URL))
  )
})

self.addEventListener('activate', (event) => {
  event.waitUntil(
    caches.keys()
      .then((keys) =>
        Promise.all(keys.filter((k) => k !== CACHE_NAME).map((k) => caches.delete(k)))
      )
      .then(() => self.clients.claim())
  )
})

self.addEventListener('fetch', (event) => {
  if (event.request.mode === 'navigate') {
    event.respondWith(
      fetch(event.request).catch(() => caches.match(OFFLINE_URL))
    )
  }
})

self.addEventListener('message', (event) => {
  if (event.data?.type === 'SKIP_WAITING') {
    self.skipWaiting()
  }
})
```
