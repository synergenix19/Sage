## Known Limitations (not blockers)

1. **EC-4 / Arabic punctuation false positive**: The `؟` (U+061F Arabic question mark) falls within `[؀-ۿ]` regex range. A message like `"Really؟ I didn't know that"` would set `code_switching=True` and fire CU-CS-001 even though no Arabic script words are present. Acceptable POC limitation.

2. **BC-3/BC-4 budget drop scenario**: When CU-SH-001 (65 words) and CU-DM-001 (55 words) both fire (running=120), CU-CS-001 (48 words) fails the budget check (120+48=168 > 150) and is silently dropped. The `word_count == 0` guard only protects the *first* rule. A code-switching user expressing shame with Khaleeji dialect loses the code-switching instruction. Acceptable for POC.

3. **ST-3 / No explicit `code_switching: False` in graph.py initial state**: LangGraph TypedDicts have no default values. `safety_check_node` always writes `code_switching` on the first pass via the returned dict, so downstream nodes always get a valid value. The `.get("code_switching", False)` fallback in `compose_prompt` also guards against this. Acceptable for POC.

4. **AC-3 / Cultural rule firing not in audit log**: `output_gate.py` logs `clinical_flags` and other state but does not log which cultural rules fired in a turn. `freeflow_respond.py` evaluates cultural rules but emits no print/log statement for `cultural_result.fired_ids`. Flag for Sprint 6 / post-POC observability.

5. **RG-1 / Em dash in PERSONA string**: The `PERSONA` constant in `freeflow_respond.py` contains an em dash inside a quoted WRONG-example (`WRONG: "That really resonates — sometimes things pile up"`). This is instructional content teaching the LLM what *not* to do. It is not an action content string and does not violate the rule-content em-dash policy.
