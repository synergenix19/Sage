**PM-3: "hook up" false positive — PASS**

`"I can hook you up with some support resources"` does NOT trigger CUO-GC-001. The phrase `"hook up"` (with space) requires both words in sequence; `"hook you up"` has an intervening word and does not match as a substring.

> *Note: This passes because Python `"hook up" in "hook you up"` evaluates to False — the words are not adjacent.*

---

**PM-4: Allowlist partial match ("patient" vs "patience") — PASS with design note**

Confirmed: `"patience"` is NOT a substring of `"be patient with yourself"`. The patterns use exact substring matching — `"patience"` requires the exact string including the `-ce` suffix.

Runtime behavior: CUO-IS-001 fires on the response `"Be patient with yourself"` when `"allah"` is in the message — because `"patience"` is not found in the response. This is **correct engine behavior**.

**Design implication:** A therapist response using `"patient"` (adjective) rather than `"patience"` (noun) will not clear the allowlist. The allowlist requires either an explicitly Islamic term or the exact word `"patience"`. This is a known trade-off for the POC — full resolution requires semantic/stemming matching, not substring.

## Warning Summary

### W1 — EF-5: Arabic normalization gap (severity: LOW)
**Location:** `src/sage_poc/rules/engine.py`, `_eval_cultural_output`  
**Issue:** Arabic-script patterns go through `p.lower()` only — `normalize_arabic()` (diacritic stripping, alef normalization) is not applied, unlike `_eval_safety` and `_eval_cultural`.  
**Impact in POC:** None. `message_en` and `response_en` are English-language fields. Arabic-script rule entries are inert for current runtime paths.  
**Production fix:** Add `normalize_arabic()` pass for Arabic-script patterns, mirroring the `_eval_safety` pattern.

### W2 — RF-6: "pig out" false positive risk (severity: LOW)
**Location:** `src/sage_poc/rules/data/cultural_output/general_cultural.json`, line 22  
**Issue:** `"pig out"` is an English idiom for overeating, not a pork-specific term. On an always-active blocklist, clinical responses about binge eating could false-trigger.
