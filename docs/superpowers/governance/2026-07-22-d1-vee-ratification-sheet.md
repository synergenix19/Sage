user's NEXT turn instead discloses harm-intrusive / OCD-veto / containment content, the screen currently

the *single* pending turn — a narrow coincidence window. We deferred the fix (a veto-marker signal into the
screen decision) and are monitoring how often the case actually occurs.

**Engineering recommendation: KEEP DEFERRED + MONITORED.** Safe by construction, narrow window, and the fix is
tracked to move up if monitoring shows it is not rare. But whether that trade-off is *clinically* acceptable
